---
title: "Mark Appointment as Attended"
api: "PUT /api/citas/:id/atender"
description: "Mark a confirmed appointment as attended"
---

## Authentication

This endpoint requires authentication and authorization.

**Required roles:** `admin` or `asistente`

**Header:**
```
Authorization: Bearer <token>
```

## Path Parameters

<ParamField path="id" type="number" required>
  The appointment ID to mark as attended
</ParamField>

## Response

Returns a success message and the updated appointment object.

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="cita" type="object">
  Updated appointment object
  
  <ResponseField name="id" type="number">
    Appointment ID
  </ResponseField>
  
  <ResponseField name="cliente_id" type="number">
    Public client ID
  </ResponseField>
  
  <ResponseField name="medico_id" type="number">
    Assigned doctor ID
  </ResponseField>
  
  <ResponseField name="fecha_solicitada" type="string">
    Requested date (ISO 8601)
  </ResponseField>
  
  <ResponseField name="hora_solicitada" type="string">
    Requested time (ISO 8601)
  </ResponseField>
  
  <ResponseField name="fecha_confirmada" type="string">
    Confirmed date (ISO 8601)
  </ResponseField>
  
  <ResponseField name="hora_confirmada" type="string">
    Confirmed time (ISO 8601)
  </ResponseField>
  
  <ResponseField name="sintomas" type="string | null">
    Patient symptoms description
  </ResponseField>
  
  <ResponseField name="estado" type="string">
    Appointment status - updated to `atendida`
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    Creation timestamp
  </ResponseField>
</ResponseField>

## Example Request

```bash
curl -X PUT https://api.example.com/api/citas/1/atender \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json"
```

## Error Responses

<ResponseField name="400 Bad Request">
  Appointment is not in confirmed status
  
  ```json
  {
    "message": "Solo se pueden atender citas confirmadas"
  }
  ```
</ResponseField>

<ResponseField name="401 Unauthorized">
  Missing or invalid authentication token
  
  ```json
  {
    "message": "Token no proporcionado"
  }
  ```
</ResponseField>

<ResponseField name="403 Forbidden">
  User doesn't have required role
  
  ```json
  {
    "message": "Acceso denegado: requiere rol admin o asistente"
  }
  ```
</ResponseField>

<ResponseField name="404 Not Found">
  Appointment not found
  
  ```json
  {
    "message": "Cita no encontrada"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Server error while updating appointment
  
  ```json
  {
    "message": "Error al marcar cita como atendida"
  }
  ```
</ResponseField>

## Notes

- This endpoint can only be used on appointments with status `confirmada`
- Attempting to mark appointments in other states (pendiente, atendida, cancelada) will result in a 400 error
- The appointment status is updated to `atendida`
- No request body is required for this operation
