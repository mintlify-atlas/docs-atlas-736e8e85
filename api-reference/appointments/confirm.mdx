---
title: "Confirm Appointment"
api: "PUT /api/citas/:id/confirmar"
description: "Confirm an appointment and assign it to a doctor"
---

## Authentication

This endpoint requires authentication and authorization.

**Required roles:** `admin` or `asistente`

**Header:**
```
Authorization: Bearer <token>
```

## Path Parameters

<ParamField path="id" type="number" required>
  The appointment ID to confirm
</ParamField>

## Request Body

<ParamField body="medico_id" type="number" required>
  The doctor ID to assign to this appointment
</ParamField>

## Response

Returns a success message and the updated appointment object.

<ResponseField name="message" type="string">
  Success message
</ResponseField>

<ResponseField name="cita" type="object">
  Updated appointment object
  
  <ResponseField name="id" type="number">
    Appointment ID
  </ResponseField>
  
  <ResponseField name="cliente_id" type="number">
    Public client ID
  </ResponseField>
  
  <ResponseField name="medico_id" type="number">
    Assigned doctor ID
  </ResponseField>
  
  <ResponseField name="fecha_solicitada" type="string">
    Requested date (ISO 8601)
  </ResponseField>
  
  <ResponseField name="hora_solicitada" type="string">
    Requested time (ISO 8601)
  </ResponseField>
  
  <ResponseField name="fecha_confirmada" type="string">
    Confirmed date (ISO 8601) - set to current date
  </ResponseField>
  
  <ResponseField name="hora_confirmada" type="string">
    Confirmed time (ISO 8601) - set to current timestamp
  </ResponseField>
  
  <ResponseField name="sintomas" type="string | null">
    Patient symptoms description
  </ResponseField>
  
  <ResponseField name="estado" type="string">
    Appointment status - updated to `confirmada`
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    Creation timestamp
  </ResponseField>
</ResponseField>

## Example Request

```bash
curl -X PUT https://api.example.com/api/citas/1/confirmar \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json" \
  -d '{
    "medico_id": 2
  }'
```

## Error Responses

<ResponseField name="400 Bad Request">
  Missing required field
  
  ```json
  {
    "message": "medico_id es obligatorio"
  }
  ```
</ResponseField>

<ResponseField name="401 Unauthorized">
  Missing or invalid authentication token
  
  ```json
  {
    "message": "Token no proporcionado"
  }
  ```
</ResponseField>

<ResponseField name="403 Forbidden">
  User doesn't have required role
  
  ```json
  {
    "message": "Acceso denegado: requiere rol admin o asistente"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Server error while confirming appointment
  
  ```json
  {
    "message": "Error al confirmar cita"
  }
  ```
</ResponseField>

## Notes

- Confirming an appointment updates its status from `pendiente` to `confirmada`
- The `fecha_confirmada` is set to the current date
- The `hora_confirmada` is set to the current timestamp
- The assigned doctor must exist in the database
