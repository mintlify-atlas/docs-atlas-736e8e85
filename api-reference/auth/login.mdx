<Note>
  Based on the source code analysis, the actual login endpoint is not directly implemented in the `/api/admin-auth` routes. 
  
  Authentication is handled through **Supabase Auth** on the client side. After successful Supabase authentication, the system uses the JWT token from Supabase to verify user permissions.
</Note>

## Authentication Flow

1. **Client-side authentication**: The application uses Supabase's authentication system to log in users
2. **Token verification**: The received JWT token from Supabase is sent in the `Authorization` header as a Bearer token
3. **Role verification**: The `authMiddleware` decodes the token and verifies the user's role in the internal database
4. **Access control**: The `requireRole` middleware checks if the user has the required role (admin or asistente)

## How to Use

### Step 1: Authenticate with Supabase

Use the Supabase client library to authenticate:

```javascript
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(SUPABASE_URL, SUPABASE_KEY)

const { data, error } = await supabase.auth.signInWithPassword({
  email: 'admin@example.com',
  password: 'your-password'
})

if (data.session) {
  const token = data.session.access_token
  // Use this token for API requests
}
```

### Step 2: Use Token in API Requests

Include the token in the Authorization header:

```bash
curl -X GET https://api.example.com/api/dashboard/stats \
  -H "Authorization: Bearer <supabase-access-token>"
```

## Token Structure

The system expects a Supabase JWT token with:
- `sub` field containing the user's Supabase UUID
- The UUID must match a user record in the internal `usuarios` table

## User Roles

The system supports the following roles:
- `admin` - Full administrative access
- `asistente` - Assistant role with limited permissions

## Related Endpoints

- See [Verify Token](/api-reference/auth/verify) for checking authentication status
- See [Get User Role](/api-reference/users/get-role) (if implemented) for retrieving user role information

## Notes

- Login is handled entirely by Supabase Auth
- The Medical Center API validates the Supabase token and checks user roles
- No password is stored or validated by this API
- Users must exist in both Supabase Auth and the internal `usuarios` table
