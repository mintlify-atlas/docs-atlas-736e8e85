---
title: 'List Contact Messages'
api: 'GET /api/contacto'
description: 'Retrieve all contact messages with client information'
---

This endpoint retrieves all contact messages submitted by public clients, ordered by creation date (newest first). Each message includes the associated client's information.

## Authentication

No authentication required based on the source code.

## Response

<ResponseField name="contactos" type="array">
  Array of contact message objects
  
  <ResponseField name="id" type="number">
    Message ID
  </ResponseField>
  
  <ResponseField name="cliente_id" type="number">
    ID of the public client who sent the message
  </ResponseField>
  
  <ResponseField name="asunto" type="string">
    Message subject or title
  </ResponseField>
  
  <ResponseField name="mensaje" type="string">
    Message content
  </ResponseField>
  
  <ResponseField name="created_at" type="string">
    Message creation timestamp
  </ResponseField>
  
  <ResponseField name="cliente" type="object">
    Associated client information
    
    <ResponseField name="cliente.nombres" type="string">
      Client's first name(s)
    </ResponseField>
    
    <ResponseField name="cliente.apellidos" type="string">
      Client's last name(s)
    </ResponseField>
    
    <ResponseField name="cliente.telefono" type="string">
      Client's phone number
    </ResponseField>
    
    <ResponseField name="cliente.email" type="string">
      Client's email address
    </ResponseField>
  </ResponseField>
</ResponseField>

## Error Responses

<ResponseField name="500 Internal Server Error" type="object">
  Returned when a server error occurs
  
  ```json
  {
    "message": "Error al listar contactos"
  }
  ```
</ResponseField>

## Example Request

```bash
curl -X GET https://api.example.com/api/contacto \
  -H "Content-Type: application/json"
```

## Notes

- Messages are ordered by `created_at` in descending order (newest first)
- The endpoint includes related client information to facilitate message management
