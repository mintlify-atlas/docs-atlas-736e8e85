---
title: 'Submit Contact Message'
api: 'POST /api/contacto'
description: 'Submit a contact message from a public client'
---

This endpoint allows public clients to submit contact messages or inquiries to the medical center.

## Authentication

No authentication required - this is a public endpoint.

## Request Body

<ParamField body="cliente_id" type="number" required>
  ID of the public client submitting the message
</ParamField>

<ParamField body="asunto" type="string" required>
  Subject or title of the message
</ParamField>

<ParamField body="mensaje" type="string" required>
  Message content or description
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message: "Mensaje enviado correctamente"
</ResponseField>

<ResponseField name="contacto" type="object">
  The created contact message object
  
  <ResponseField name="contacto.id" type="number">
    Message ID
  </ResponseField>
  
  <ResponseField name="contacto.cliente_id" type="number">
    ID of the public client
  </ResponseField>
  
  <ResponseField name="contacto.asunto" type="string">
    Message subject
  </ResponseField>
  
  <ResponseField name="contacto.mensaje" type="string">
    Message content
  </ResponseField>
  
  <ResponseField name="contacto.created_at" type="string">
    Creation timestamp
  </ResponseField>
</ResponseField>

## Error Responses

<ResponseField name="400 Bad Request" type="object">
  Returned when required fields are missing
  
  ```json
  {
    "message": "cliente_id, asunto y mensaje son obligatorios"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error" type="object">
  Returned when a server error occurs
  
  ```json
  {
    "message": "Error al enviar mensaje"
  }
  ```
</ResponseField>

## Example Request

```bash
curl -X POST https://api.example.com/api/contacto \
  -H "Content-Type: application/json" \
  -d '{
    "cliente_id": 45,
    "asunto": "Consulta sobre horarios de atención",
    "mensaje": "Buenos días, quisiera saber si atienden los días sábados y cuál es el horario. Gracias."
  }'
```

## Example Response

```json
{
  "message": "Mensaje enviado correctamente",
  "contacto": {
    "id": 78,
    "cliente_id": 45,
    "asunto": "Consulta sobre horarios de atención",
    "mensaje": "Buenos días, quisiera saber si atienden los días sábados y cuál es el horario. Gracias.",
    "created_at": "2026-03-05T14:45:00.000Z"
  }
}
```

## Notes

- The `cliente_id` should correspond to an existing public client record in the database
- Contact messages are typically reviewed by administrative staff
- Messages are stored with timestamps for tracking and follow-up purposes
